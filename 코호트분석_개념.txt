### 코호트 분석
동일한 경험 & 동일한 기간
=> 서울초 = 코호트 X
=> 서울초 2002년도 졸업생 = 코호트 O
(※ 주의 : 코호트는 비교를 전제로 함)

- 연 단위 코호트 : 연도 단위로 비교
- 월 단위 코호트 : 월 단위로 비교

코호트에 속한 사람은 동일한 이벤트를 경험해야함

ex. 7일짜리 코호트라고 가정했을 때(이벤트 기간이 7일)
폭망 기간이 2일이고 정상 서비스 기간이 5일이면 해당 코호트는 7일짜리 코호트 X
=> 2일을 제외하고 5일짜리 코호트를 둬야함 & 비교군도 5일짜리 코호트를 만들어서 비교

# 코호트 분석 방법
1. 기간으로 비교
2. 같은 기간동안 서로 다른 경험을 한 사람들간 비교(매장 출입을 앞문 vs 뒷문)

세그먼트 : 고객을 나눈다는 말(군집화)
=> 신용카드로 결제한 고객 vs 신용카드로 결제하지 않은 고객
=> 50만원 이상 구매한 고객 vs 50만원 미만 구매한 고객

* 접속 국가로 세그먼트를 나누는 것은 같은 경험이 아니므로 코호트 X
=> 접속 국가 : 코호트를 나누는 기준이 될 수 없음

* 성별 & 나이 : 코호트 구분 기준이 될 수 없음
즉, 인구통계는 세그먼트를 나누는 속성임

# 코호트 = 동일한 기간에, 동일한 경험을 한 사람끼리 묶는 것
# 세그먼트 = 전체 고객(집합)을 특정 기준으로 나눔(하나 이상의 속성을 공유한 사람들의 집합을 의미)

## 코호트를 세그먼트로 나누는 방법 :
1. 팝업스토어 방문 -> 2. 결제방법으로 구분
=> 1. 구매여부 -> 2. 현금/카드/기타 결제방법

* 11월 3주차 내방객 코호트
1. (1) 비구매고객 (2) 구매고객
2. (1) 비구매고객 (2-1) 신용카드 결제 (2-2) 현금 결제 (2-3) 기타 결제 => 총 4개의 세그먼트
=> 4개의 세그먼트에 대해 "객단가 살펴보기" or "재구매율 보기"
=> 구매고객에 대해 더 많은 정보를 얻을 수 있음

## 세그먼트를 코호트로 나누는 방법 :
지난달 전체 고객 중 신용카드 결제자로 구분
=> 11월 신용카드 결제 고객 -> 해당 세그먼트를 주차별로 구분
(1) 11월 1주차
(2) 11월 2주차
(3) 11월 3주차
(4) 11월 4주차
=> 11월에 신용카드 결제 고객 중 특정 주차에 방문해서 구매한 사람들의 금액이나 구매 건수를 볼 수 있음

## 결론 :
- 세그먼트가 큰 의미이고, 코호트가 작은 의미임
- 코호트를 세분화하면 할수록 데이터 수집량이 많아지고, 그만큼 분석도 어려워짐(경우의 수 증가)
- 어떤 이벤트의 효과를 측정하고자 한다면 미리 어떤 데이터를 어떤 크기(해상도)로 측정하고자 하는지를 정해야함
=> 뒤늦게 분석하려고 보니 수집된 데이터가 없거나 분석 단위가 맞지 않을 수 있음
